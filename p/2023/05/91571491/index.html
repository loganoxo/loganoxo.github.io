    # 1 简介

    • 直接 vim

    # 1⃣️ normal模式

    • 快速移动到上次编辑位置，当光标移动到其他位置后，可以使用 gi 快速进入标记模式，并且光标位置是上一次编辑的位置

    # 2⃣️ 编辑模式

    • i （insert） a（append，光标后一个字符输入）o（open a line below 当前行下面插入新一行并且光标移入新插入的空白行）

    • I 当前行最前面输入 ， A 当前行末尾输入 ，O 当前行上面插入一行并且光标移入新插入的空白行

    • esc 退回normal模式，ctrl+c ctrl+[ 也可以

    • ctrl+h 删除上一个插入的字符

    • ctrl+w 删除上一个插入的单词

    • ctrl+u 删除当前行

    # 3⃣️ 命令模式

    • :wq! 保存并强制退出 w保存 q推出 ！强制
    • 全局替换， :% s/foo/bar/g 把foo全局替换为bar
    • 设置行号 ， :set nu
    • 语法高亮 ，:syntax on
    • 搜索结果高亮，:set hls
    • 边搜索边高亮，:set incsearch
    • 文本自动对齐，:set autoindent

    # 4⃣️ 可视化模式

    • normal模式下使用

    • 使用 v 进入visual模式，从光标的位置开始用方向键接连选择文本

    • 使用 V进入visual模式，默认选中当前行，用方向键接连选择上面或者下面的行，只能选整行

    • 使用 control+v进入visual模式，可以从光标位置任意选择

    • 选中之后可以进行批量操作，比如选中之后，按下 d，则可删除选中的所有文本

    # 3 启动vim

    • vim 打开启动页面
    • vim -c cmd file ，在打开文件前，先执行指定的命令；
    • vim -r file , 恢复上次异常退出的文件；
    • vim -R file ，以只读的方式打开文件，但可以强制保存；
    • vim -M file ，以只读的方式打开文件，不可以强制保存；
    • vim -y num file ，将编辑窗口的大小设为num行；
    • vim + file ， 从文件的末尾开始；
    • vim +num file ，从第num行开始,+也要；
    • vim +/string file ，打开file，并将光标停留在第一个找到的string上。
    • vim –remote file ， 用已有的vim进程打开指定的文件。 如果你不想启用多个vim会话，这个很有用。但要注意， 如果你用vim，会寻找名叫VIM的服务器；如果你已经有一个gvim在运行了， 你可以用gvim –remote file在已有的gvim中打开文件。

    # 4 启动多个vim 标签或者窗口

    • :sp 和 :vs 将窗口分割为4个，这时四个窗口都是相同的文件，即同一个文件buffer
    • 或者 通过 :sp 和 :vs + 文件名 ，可以新建窗口同时打开文件
    • 通过 :b 切换 ，或者 :e 新建 buffer ，这样就在每个窗口可以打开不同的文件
    • 通过 <ctrl+w>+方向键 切换窗口 或者移动窗口
    • :e file ，关闭当前编辑的文件，并开启新的文件。 如果对当前文件的修改未保存，vi会警告。
    • :e! file ，放弃对当前文件的修改，编辑新的文件。
    • :e+file ， 开始新的文件，并从文件尾开始编辑。
    • :e+n file ， 开始新的文件，并从第n行开始编辑。
    • :enew ，编译一个未命名的新文档。(CTRL-W n)
    • :e ， 重新加载当前文档。
    • :e! ， 重新加载当前文档，并丢弃已做的改动。
    • :e#或ctrl+^ ， 回到刚才编辑的文件，很实用。
    • :f或ctrl+g ， 显示文档名，是否修改，和光标位置。
    • :f filename ， 改变编辑的文件名，这时再保存相当于另存为。
    • gf ， 打开以光标所在字符串为文件名的文件。
    • :w，保存修改。
    • :n1,n2w filename ， 选择性保存从某n1行到另n2行的内容。
    • :wq ， 保存并退出。
    • ZZ ，保存并退出。
    • :x ， 保存并退出。
    • :q[uit] ，退出当前窗口。(CTRL-W q或CTRL-W CTRL-Q)
    • :saveas newfilename ，另存为
    • :browse e ， 会打开一个文件浏览器让你选择要编辑的文件。 如果是终端中，则会打开netrw的文件浏览窗口； 如果是gvim，则会打开一个图形界面的浏览窗口。 实际上:browse后可以跟任何编辑文档的命令，如sp等。 用browse打开的起始目录可以由browsedir来设置：
      • :set browsedir=last ， 用上次访问过的目录（默认）；
      • :set browsedir=buffer ，用当前文件所在目录；
      • :set browsedir=current ， 用当前工作目录；

    # 6 翻屏

    # 1⃣️ 单词

    • w/W 移动到下一个 word/WORD 开头。 e/E 下一个 word/WORD结尾
    • b/B 回到上一个word/WORD开头，可以理解为backword
    • 前面可以加上数字，表示命令重复几次，比如3w等同于按下三次w
    • word指的是以非空白符分割的单词，WORD以空白符分割的单词 例如 List是一个word，List<AuditItemVO.AuditItem>是一个Word

    # 2⃣️ 当前行快速移动光标

    • $ 移动到行的末尾
    • 0 移动到行的开头
    • ^ 移动到第一个非空白字符
    • g_ 移动到最后一个非空白字符

    # 3⃣️ 当前行搜索

    • 同一行快速移动的方式其实是搜索一个字符并且移动到该字符
    • normal模式下使用 f+char 可以移动到char字符上，t移动到char的前一个字符 如果第一次没搜到，可以用 分号/逗号 继续搜该行下一个/上一个
    • F 表示反过来搜前面的字符

    # 4⃣️ 行间跳转

    • : 行号 跳转到对应行
    • gg 移动到第一行
    • G 移动到最后一行
    • ctrl + I 下次光标位置 ctrl + o上次光标位置
    • 跳转多行 数字+Ctrl+d 向下移动 数字+Ctrl+u 向上移动
    • 跳转到屏幕开头 H head
    • 跳转到屏幕中间 M middle
    • 跳转到屏幕结尾 L lower
    • 上下翻页 Ctrl+u/ctrl+f （upword/forward）
    • 把屏幕置为中间 zz
    • ( 前移1句。
    • ) 后移1句。
    • { 前移1段。
    • } 后移1段。
    • m a 标记当前行为a，单引号+a即可跳转

    # 5⃣️ 搜索和替换

    ​ 搜索

    • 使用/或者？进行前向或者反向搜索
    • 使用n/N跳转到下一个或者上一个匹配

    ​ 替换

    • substitute 命令允许我们查找并且替换掉文本,并且支持正则式

    • :[range]空格s[ubstitute]/{pattern}/{string}/[flags]

    • range表示范围比如:10,20表示10-20行,%表示全部 pattern是要替換的模式, string是替換后文本

    • Flags有几个常用的标志 g( global)表示全局范围内执行 c( confirm)表示确认,可以确认或者拒绝修改 n( number)报告匹配到的次数而不替換,可以用来查询匹配次数

    # 6⃣️ 快速删除

    • v模式下通用
    • Vim在normal模式下使用x快速删除一个字符
    • 删除单词包括空格daw（d around word)，或者删除单词不包括空格 diw,
    • dw删除光标到单词尾(包括空格)
    • d和ⅹ都可以搭配数字来执行多次
    • 删除一行 dd，删除两行 2dd
    • dt ，delete to，删除直到，dt) 删除括号前的内容 ——当前行
    • d0， 删除光标位置到行头
    • d$，删除光标位置到行尾

    # 7⃣️ 快速修改

    • 常用有三个，r（replace），c（change），s（substitute）
    • normal模式下使用r可以替换一个字符。s删除当前字符并进入插入模式
    • R 当前行光标后每输入一个字符都进行替换
    • 使用c配合文本对象，我们可以快速进行修改， cw删除当前单词并进入插入模式

    ctrl+m

    # 9⃣️ 快速跳转到匹配的括号

    %

    # 8 重要操作

    • 如果你学过主流的编程语言,一定知道面向对象编程

    • Vim里文本也有对象的概念,比如一个单词,一段句子,一个段落

    • 很多其他编辑器经常只能操作单个字符来修改文本,比较低效

    • 通过操作文本对象来修改要比只操作单个字符高效

    • [number]<command>[text object]

    • number 表示次数, command 是命令, d(delete), c (change), y (yank 复制)，v (visual 选中)

    • text object 是要操作的文本对象 , 比如单词 w word, 句子 s sentence, 段落 p paragraph

    # 10 vim复制粘贴和寄存器

    • 宏可以看成是一系列命令的集合
    • 我们可以使用宏『录制』一系列操作 , 然后用于『回放』
    • 宏可以非常方便地把一系列命令用在多行文本上
    • normal模式下 ， Vim使用 q 来录制 , 同时也是 q 结束录制
    • 使用 q register_name 选择要保存的寄存器 , 把录制的命令保存其中 使用 @ register_name 回放寄存器中保存的一系列命令

    • 先给一行加上双引号,然后再回放到其他所有行

    • 我们先使用 qa 开始录制到a寄存器 , 给一行加上双引号 , 之后使用 q 退出

    • 在剩下的所有行中 用 @a回放录制的宏

    • 直接处理多行： v 进入v模式，选中行，:normal @a

    • 还可以直接不用宏，直接用 v模式操作，在v模式下选中要操作的行，然后 :normal I " :normal A"

    # 12 vim补全

    • 使用ctr+n和ctr+p补全单词
    • 使用ctrl+x ctrl+f 补全文件名
    • 使用ctrl+ x ctrl+o补全代码,需要开启文件类型检查,安装插件

    # 13 vim更换配色

    • 使用 :colorscheme 显示当前的主题配色,默认是 default
    • 用 :colorscheme <ctrl+d> 可以显示所有的配色
    • 有中意的配色后,用 :colorscheme 配色名就可以修改配色
    • 从网络上寻找更好看的配色 https://github.com/flazz/vim-colorschemes 安装之后就有大量的主题可以更换了
